Title_[ 포트폴리오 모음 페이지 ] 단일 페이지 HTML5 / CSS3 / JavaScript(& Jquery) 메인컬러 :
두번째 작품
( 모바일_768px이하 / 태블릿_1024이하 / PC_1280이하 / PC_1600이하 / PC_1600 이상)
미디어 쿼리를 통한 분기별 반응형 웹. 디자인 작업 포함.
크롬 브라우저와 1920x1080 해상도에 최적화.
순서상으론 두번째지만 가장 중요한 페이지므로 첫번째에 두었다.
최대한 플러그인이나 라이브러리는 배제하면서도 이것저것 효과만 잔뜩 넣어보긴 했는데... 난잡한 느낌이다.
근데 유독 모바일에서 퍼포먼스가 상당히 떨어지는데 이유를 잘 모르겠다.이미지를 큰거로 사용해서??

디자인 작업은 최소화하여 포트폴리오 화면별 이미지정도만 캡쳐하거나 수정했고 그 외에는 죄다 코딩과 딩벳폰트 위주로 구현.
그러다 보니 단순한 도형이나 텍스트 위주가 돼버렸다.
사실 처음에는 단순하고 깔끔하게 서체와, 폰트 사이즈, 두께정도만 조절해가면서 제목과 작품목록 정리한 것만 넣을 생각이었는데, 갑자기 애니메이션 효과에 꽂혀서 쓰잘데기 없이 많이 집어 넣었다.

keyframes와 Jquery, animation, transition을 활용한 애니메이션과 svg를 활용한 벡터그리기 효과 등등, 직접 코드를 쓸 수 있는 선에서만 섞어봤다.
정작 이렇게 꾸미기 위주로 하다보니 시맨틱구조와 아웃라인은 그렇다치고 웹 접근성(Accessibility) 측면에선 미흡해진 것 같다.개인적인 생각에
이 쯤부터 웹호환성(크로스브라우징), 웹표준, 웹접근성 등을 위시하여 CSS 네이밍 방법론BEM,SMACSS,OOCSS 등에 관심을 가졌지만 현실적인 생각포트폴리오에서 굳이? 내지 아이폰의 부재 등)에 그냥 개념정도로만 잡고 작업에 적용하진 않았다.
어차피 클래스 네이밍같은 것은 회사자체 규정이 따로 있는 경우가 많다고 하기도 하고. 아직은 그냥 '이런것이 있다'정도만 알아도 되겠다 싶어서 보류.
퍼블리싱 스킬부터 갈고 닦자!
대신 기존에 주로 사용하던 CSS속성선택자는 거의 제외하고 웬만하면 id나 class선택자로 변경했으며, 보통 해당 클래스만 선택해서 앞에 탭을 띄워주는 것으로 구조를 파악할 수 있게 하던 방식에서 앞에 부모 요소를 잔뜩 붙여주는 방식으로 변경했다.나름 각각의 장단점이 있는 듯 싶다.
물론 진정한 웹 퍼블리셔가 되려면 퍼블리싱 스킬, 웹호환성, 웹표준, 웹접근성부터 챙겨야 겠지만 아직은 미숙하다보니...
솔직히 ui디자인같은 것은 일일이 짜는 것보다는 jquery UI같은 플러그인이나 부트스트랩 같은 오픈소스 프레임워크를 다루는 것이 낫다고는 하는데 아직 저런 것들을 이용하기 보다는 공부도 하면서 숙달도 할겸, 모두 날코딩하는 것이 좋아보여서 계속 이렇게 작업할 예정.

물론 디자인적인 작업은 지양하고 추후 작업 부터는 무료배포하는 psd나 웹디자인 템플릿, 혹은 기존에 존재하는 다른 사이트의 이미지 소스만 가져와서 마크업 위주로 해봐야겠다.

Title_[ 놀이넷 ] 메인페이지 & 첫번째 서브페이지까지 HTML5 / CSS3 / JavaScript(& Jquery) 메인컬러 :
첫번째 작품
( 모바일_768px이하 / 태블릿_1024px이하 / PC_1024px이상 )
미디어쿼리를 통한 분기별 반응형 웹. 디자인 작업 포함.
단순 예제나 원페이지가 아니라 처음부터 끝까지스토리보드 스케치, 로고&아이콘 디자인, 플로우차트, 스타일 가이드, 등등 학원에서 배운 실무의 흐름대로 차근차근 과정을 밟아 제작하려 했으나...
사실 구상은 잔뜩 해놨는데 중간에 김이 새서디자인 작업만 잔뜩 & 주제가 별로 마음에 안듬 정말 기초적인 수준으로만 제작했다. 그래도 이 사이트를 만들면서 기획과 디자인의 중요성을 톡톡히 알았다. 사이트에 들어갈 컨텐츠를 짜내는 것과 디자인을 고민하는 게 힘든 일라는 걸 깨닫는 계기가 되었다.
포괄적으로 배우면서, 디자인은 어렵고 관심도 그냥저냥이지만 코딩은 재밌었다.
물론 자바스크립트 부분이래봐야 기본적인 부분을 제외하면 간편한 라이브러리인 제이쿼리만 사용했으며, 그마저도 GNB는 죄다 :hover로 처리해서 주된 기능은 슬라이드 뿐이긴 하다.

그래도 정말 단순하게 이동만 시키는 코딩에서 좀더 논리적으로 구성할 수 있을까 고민하는 과정에서 많이 부족하다는 걸 느낄 수 있었다. 배울 것이 너무나도 많고 멀리 있다는 생각을 가질 수 있었다.
여담으로 디자인이라든지 마크업한 분량이 적어서 넣지 말까 생각했지만 나름 첫 작업물인 만큼 기념으로 등재했다.
